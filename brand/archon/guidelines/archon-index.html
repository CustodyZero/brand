How It Works

Three layers. No ambiguity.

01 — Capability
What agents are allowed to do

Capability modules define the outer boundary of what any agent in your system can do. Nothing outside an enabled capability module can execute. Period.

No capability is active without explicit operator enablement. The default state is restricted.
02 — Restriction
What is currently permitted

Dynamic restriction rules define what's permitted right now, within the capability boundary. Rules are explicit, inspectable, and hashed. No rule widens authority implicitly.

Every rule set is evaluated against a hashed snapshot. If the snapshot doesn't match, execution stops.
03 — Execution
What actually runs

Only actions that pass deterministic validation against both capability modules and restriction rules reach execution. Archon doesn't reason about policy. It enforces it.

Every decision is logged. Every enforcement event is inspectable. Nothing runs invisibly.

Capabilities

What Archon enforces.

CAP

Capability Modules

Define the outer boundary of what agents can do. Modules are operator-enabled explicitly. The default state is restricted. Nothing executes outside an active module.

RULE

Rule Enforcement

Dynamic restriction rules evaluated deterministically at every execution gate. Rules are hashed. Snapshot mismatches halt execution. No implicit authority.

LOG

Inspectable Decisions

Every enforcement decision is logged with full context. Every validated or rejected action is auditable. Nothing runs invisibly. The system is always explainable.

LOCAL

Local First

Archon runs on your hardware. No cloud dependency, no external enforcement service, no phone-home. Your agent stack, your rules, your infrastructure.

HASH

Rule Snapshots

Every rule set is evaluated against a cryptographic hash of the current snapshot. Rules can't be silently modified. Integrity is enforced at evaluation time.

API

Agent Agnostic

Archon coordinates agents, not frameworks. Bring your existing agent setup. Archon sits as the enforcement layer between your agents and execution.
