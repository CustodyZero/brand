01 — The Name
Type

Four letters. A verb: to type, to put words on a page. A noun: a type of thing, a category, a classification. A reference to typography itself — the art of setting words so they can be read.

The name is the act. Not the tool, not the technology behind it. Type is what you do when you sit down to write.

02 — Positioning
What Type Is

Type is a writing instrument. Not an AI product. Not a note-taking app. Not a word processor with a chatbot bolted on.

It is a page. A good page — the kind where you forget you're using software and remember you're writing. The LLM is there the way a sharp editor is there: present when needed, invisible when not, never the one holding the pen.

Type is local-first. Your words live on your machine. Your projects are yours. The tool is opinionated about how writing should feel, and indifferent to how other people think it should work.

The positioning test: does this feature serve the person writing, or does it serve the technology? If it serves the technology, it doesn't ship.
03 — Voice & Tone
How Type Speaks

Type speaks like a well-made object. Quietly. It doesn't explain itself, promote itself, or congratulate you. It communicates only when communication is necessary, and then with precision.

04 — Color System
Ink and Paper

The palette is paper and ink. Warm, not clinical. Color appears only when it carries meaning: the editor's pencil for focus and attention, assistant ink for AI-contributed content. Everything else is surface and text.

Paper is the ground. The page floats on it, slightly brighter.
No pure white (#FFFFFF). No pure black (#000000). Warmth everywhere.
No gradients. No drop shadows except the page itself (subtle, structural).
Pencil is earned — it marks focus, active cursors, margin lines. Sparingly.
Assist is informational, never alarming. AI text is a different ink, not a warning.
05 — Typography
The Manuscript

Three faces, each with a clear role. Cormorant for display and the wordmark — high contrast, calligraphic warmth. Literata for the writing surface — designed for sustained reading, optical sizes that adapt. DM Mono for chrome — quiet, geometric, stays out of the way.

These are Type’s voice. The user’s own writing will use their chosen fonts.

06 — Spatial Principles
The Page Is the Interface

The default state of Type is a page. A single column, approximately 65–75 characters wide, centered on the screen. The background is paper; the page is slightly brighter paper. There is a faint margin line on the left, like a ruled notebook.

Everything else — project navigation, document structure, AI assistance, search, export — lives at the periphery and appears only when summoned.

sidebar · summoned
margins · annotations
The page width (680px / ~65–75ch) is a typographic constraint, not a layout decision. It ensures comfortable reading at body text sizes. The periphery exists outside this column.
07 — Interaction Model
Keyboard-First, Mouse-Accessible

Every peripheral layer is reachable by keyboard and by mouse gesture. The writer’s hands should not need to leave the keyboard to access any feature. But the mouse can summon the same layers through edge proximity or direct click.

Command Palette
Global access to every action. Search projects, invoke AI, change export format, navigate structure.
&Cmd;K keyboard
Project Sidebar
Document tree, project files, recent work. Slides in from the left edge.
&Cmd;1 mouse → left edge

Contextual AI assistance anchored to the current selection or cursor position. Appears in the right margin.

Search across all your writing by meaning, not just keywords. Surfaces related passages from other projects.

08 — Motion
Quiet, Purposeful, Physical

Motion in Type serves two purposes: it communicates state transitions, and it gives the interface a sense of physical weight. Panels slide, they don’t pop. Text fades in, it doesn’t appear. The cursor blinks like a heartbeat.

Nothing bounces. Nothing overshoots. Nothing calls attention to itself.

Cursor
1s ease, infinite. The only perpetual animation. Pencil red.
Panel Reveal
500ms ease-out. Slides from edge. Content fades in 200ms after panel settles.
Content Stagger
800ms per item, 80–100ms stagger. For lists, search results, outline items.
AI Text Arrival
Character-by-character in assist ink. Not instant. The machine is writing, visibly.
perhaps consider the inverse…
Easing: cubic-bezier(0.16, 1, 0.3, 1) for reveals (fast start, gentle settle). cubic-bezier(0.65, 0, 0.35, 1) for state transitions. No linear easing anywhere — everything has weight.
09 — What Type Is Not
Boundaries
— Not a chat interface. The page is primary. The LLM is peripheral.
— Not a note-taking app. Type is for long-form writing — essays, articles, chapters, books.
— Not a word processor. No ribbon bars, no formatting toolbars, no template galleries.
— Not a cloud product. Your words live on your machine. The tool is local-first.
— Not an AI writing tool. You write. The AI assists. The distinction is not cosmetic.
— Not trying to be everything. Thirty features done well. Not three hundred done carelessly.
10 — The Test
One Question

Does this feel like sitting down at a good desk with a blank page and a sharp pencil?

If yes, proceed. If it feels like opening an app, reconsider.
